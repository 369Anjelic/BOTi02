================================================================================
                   🧠 PROMPT ENGINEERING - BEST PRACTICES
================================================================================

REGEL 1: BE SPECIFIC & CLEAR
════════════════════════════════════════════════════════════════════════════════

❌ FALSCH:
"Write code"

✅ RICHTIG:
"Write a Python function that takes a list of numbers and returns the 
sum of all even numbers. Include error handling for non-numeric values.
Return as a complete, tested function."


REGEL 2: GIVE CONTEXT & CONSTRAINTS
════════════════════════════════════════════════════════════════════════════════

❌ FALSCH:
"Explain machine learning"

✅ RICHTIG:
"Explain machine learning in 3 paragraphs for a high school student.
Focus on: what it is, how it's used, why it matters. Use real examples."


REGEL 3: USE ROLE-BASED PROMPTING
════════════════════════════════════════════════════════════════════════════════

❌ FALSCH:
"Help me debug this code"

✅ RICHTIG:
"You are an expert Python debugger with 10 years experience.
Here's code that's not working. Find the bug(s), explain why it breaks,
and provide a fix with explanation."


REGEL 4: PROVIDE EXAMPLES (Few-Shot)
════════════════════════════════════════════════════════════════════════════════

❌ FALSCH:
"Convert text to JSON"

✅ RICHTIG:
"Convert the following text to JSON format.

Example:
Input: "Name: Alice, Age: 25, City: NYC"
Output: {"name": "Alice", "age": 25, "city": "NYC"}

Now convert this:
[YOUR TEXT HERE]"


REGEL 5: CHAIN-OF-THOUGHT - STEP BY STEP
════════════════════════════════════════════════════════════════════════════════

❌ FALSCH:
"Is this code correct?"

✅ RICHTIG:
"Let's analyze this code step by step.
First, what does each function do?
Second, trace through with example input.
Third, identify any edge cases or bugs.
Finally, provide the fixed version."


REGEL 6: OUTPUT FORMAT SPECIFICATION
════════════════════════════════════════════════════════════════════════════════

❌ FALSCH:
"Create API documentation"

✅ RICHTIG:
"Create API documentation in this format:
[endpoint_name]
- Method: GET/POST/etc
- URL: /api/path
- Parameters: {...}
- Response: {...}
- Example: curl ..."


REGEL 7: CONSTRAINTS & QUALITY
════════════════════════════════════════════════════════════════════════════════

❌ FALSCH:
"Write a tutorial"

✅ RICHTIG:
"Write a tutorial for beginners on Django models with:
- 500-800 words
- 3-4 code examples (complete, runnable)
- Common mistakes section
- Practice exercises
- Markdown format with code blocks"


REGEL 8: SYSTEM PROMPTS vs USER PROMPTS
════════════════════════════════════════════════════════════════════════════════

SYSTEM PROMPT (behavior, personality):
"You are a helpful Python tutor. Explain concepts clearly with examples.
Always check for understanding. Be encouraging."

USER PROMPT (specific task):
"Teach me about decorators in Python."


REGEL 9: TEMPERATURE & CREATIVITY
════════════════════════════════════════════════════════════════════════════════

temperature: 0.0     → Deterministic, factual (code, facts)
temperature: 0.5    → Balanced (explanations)
temperature: 1.0    → Creative (stories, brainstorming)


REGEL 10: TOKEN MANAGEMENT
════════════════════════════════════════════════════════════════════════════════

Short context → Faster responses (better!)
Long context  → More expensive (use only when needed!)

OPTIMIZATION:
- Remove unnecessary context
- Summarize instead of full text
- Use chunking for large files
